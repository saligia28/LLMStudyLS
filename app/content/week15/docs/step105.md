# Step 105: 模型量化｜整理专题

## 学习目标

这是 Week 15 的收尾，也是 Week 11-15 这条阶段性学习路径的总复盘。

完成后你应该能：

1. 用一张图串起 RAG、应用落地、本地部署和量化
2. 清楚说明 DeepSeek 主线与本地选修轨的关系
3. 根据真实约束做技术选择
4. 知道下一阶段还可以往哪里继续深入

---

## 一、Week 11-15 的新全景图

```text
必修主线
文档
  → Chunking
  → Local Embedding
  → Vector Store
  → Query Rewrite / Rerank（DeepSeek）
  → DeepSeek Answer
  → Web App

进阶选修
  Ollama：本地替代与离线开发
  vLLM：高并发部署
  Quantization：资源受限下的模型切换
```

这五周现在应该被理解成：

- Week 11-12：主线产品能力
- Week 13-15：本地部署与推理优化选修

## 三、决策树（DeepSeek-first）

```text
你的目标是什么？
│
├── 先把产品跑起来
│   └── DeepSeek + 本地 embedding
│
├── 数据必须留在本地
│   └── Ollama 或 vLLM + 本地 embedding
│
├── 并发量很高
│   └── DeepSeek 主线之外，再评估 vLLM
│
└── 机器资源有限
    └── 量化模型作为替代路径
```

---

## 四、这阶段最重要的认知变化

1. **RAG 不是“一个模型做所有事”**  
   主线应该明确分成：检索底座和生成链路。

2. **DeepSeek 是当前必修主线，不是所有能力都由它承担**  
   它负责回答、改写、rerank、工具调用；embedding 另走本地服务。

3. **Ollama / vLLM / 量化更适合被理解成部署和运行策略**  
   它们重要，但不是继续学主线之前的前置条件。

---

## 五、小结

Week 11-15 现在可以被更准确地概括成一句话：

**先用 DeepSeek + 本地 embedding 做出稳定主线，再按需要进入本地部署和量化选修。**

只要你把这条线抓牢，后面无论接 Agent、多模态还是部署优化，都不会再把能力边界写混。
