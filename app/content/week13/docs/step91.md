# Step 91: Ollama 入门｜整理笔记

> **定位**：Week 13 属于进阶选修，服务于“本地部署 / 离线运行 / 本地替代”场景，不是继续学习 DeepSeek 主线的前置条件。下文提到的模型标签仅作示例，请以 Ollama 官方模型库当前稳定标签为准。

## 学习目标

这一节是 Week 13 的收尾，帮助你把 Ollama 放回整个学习路径里的正确位置。

完成后你应该能：

1. 总结 Ollama 的核心能力和局限
2. 区分它和 DeepSeek 主线的关系
3. 明白什么时候该把它作为本地替代路径
4. 为 Week 14 的 vLLM 选修轨做好准备

---

## 一、Ollama 在这套课程里的位置

当前推荐的理解方式是：

```text
主线：DeepSeek + 本地 embedding
选修 1：Ollama（本地替代 / 离线开发）
选修 2：vLLM（高并发部署）
选修 3：Quantization（运行时优化）
```

也就是说，Ollama 很重要，但它不是继续学 RAG 主线前必须先跨过的一关。

---

## 二、Ollama 的核心价值

| 场景 | 推荐程度 | 理由 |
| --- | --- | --- |
| 个人开发 / 原型 | 很高 | 本地零服务依赖 |
| 隐私敏感数据 | 很高 | 数据不出机器 |
| 离线演示 | 很高 | 不依赖外网 |
| 生产级高并发 | 一般 | 不如 vLLM |
| 继续学 DeepSeek 主线 | 选修 | 不是前置条件 |

---

## 三、和主线的关系

如果你已经在主线上完成了：

- DeepSeek 聊天 / JSON 输出 / Function Calling
- 本地 embedding + RAG
- Web 应用落地

那么学习 Ollama 的价值主要是：

1. 理解本地模型的运行方式
2. 为离线场景准备替代路径
3. 为后面的量化和 vLLM 铺路

---

## 四、小结

Week 13 最重要的结论不是“学会某个具体模型名”，而是：

1. Ollama 是本地替代路径
2. 它服务于隐私、离线、成本等额外约束
3. 它和 DeepSeek 主线是互补关系，不是先后依赖
