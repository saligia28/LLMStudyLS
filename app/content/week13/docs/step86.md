# Step 86: Ollama 入门｜运行 llama3 / qwen2 模型

## 学习目标

这一节的重点是真正把两个主流开源模型跑起来，建立对模型「个性」的直观感受。

做完本节后，你应该能：

1. 拉取并运行 `llama3:8b` 与 `qwen2:7b` 两个模型
2. 在终端中进行完整的多轮对话
3. 理解 GGUF 格式与量化级别的含义
4. 用 `ollama show` 读懂模型卡片：参数量、context length、量化方式
5. 对比 llama3 与 qwen2 在中文任务上的实际表现差异

> **核心**：模型不只是参数量的数字，它有语言偏好、训练数据偏向与指令跟随风格。亲自跑一遍，才能形成判断。

## 一、拉取两个模型

### 1.1 拉取 llama3:8b

```bash
ollama pull llama3:8b
```

Meta 的 Llama 3 系列，8B 参数版本，英文能力强，中文基础可用。模型大小约 4.7 GB（Q4_0 量化）。

### 1.2 拉取 qwen2:7b

```bash
ollama pull qwen2:7b
```

阿里巴巴的 Qwen2 系列，7B 参数版本，专为中英双语优化，中文表达更自然流畅。模型大小约 4.4 GB。

### 1.3 确认下载成功

```bash
ollama list

# 预期输出：
# NAME          ID              SIZE    MODIFIED
# llama3:8b     365c0bd3c000    4.7 GB  3 minutes ago
# qwen2:7b      dd314f039b9d    4.4 GB  1 minute ago
```

## 二、理解 GGUF 格式与量化

### 2.1 什么是 GGUF

GGUF（GPT-Generated Unified Format）是 llama.cpp 项目定义的模型文件格式。它的特点是：

- **单文件打包**：所有权重、tokenizer、配置放在一个 `.gguf` 文件里
- **支持量化**：原始 FP32 权重压缩为更低精度（Q4、Q8 等），大幅减小体积
- **跨平台**：同一个文件在 CPU、Apple Silicon、NVIDIA GPU 上都能运行

### 2.2 量化级别对比

| 量化方式 | 每参数位数 | 7B 模型大小 | 质量损失 | 推荐场景 |
|---|---|---|---|---|
| Q2_K | 2 bit | ~2.8 GB | 明显 | 极度内存受限 |
| Q4_0 | 4 bit | ~4.1 GB | 轻微 | 日常使用默认值 |
| Q4_K_M | 4 bit（混合） | ~4.4 GB | 极轻微 | 质量与速度平衡 |
| Q8_0 | 8 bit | ~7.7 GB | 几乎无 | 内存充足时优选 |
| F16 | 16 bit | ~14 GB | 无 | 需要最高精度 |

> Ollama 默认拉取的版本通常是 `Q4_0` 或 `Q4_K_M`，适合大多数本地使用场景。

### 2.3 用 ollama show 查看量化信息

```bash
ollama show llama3:8b
```

```text
  Model
    architecture        llama
    parameters          8.0B
    context length      8192
    embedding length    4096
    quantization        Q4_0

  License
    META LLAMA 3 COMMUNITY LICENSE AGREEMENT
```

```bash
ollama show qwen2:7b
```

```text
  Model
    architecture        qwen2
    parameters          7.6B
    context length      32768
    embedding length    3584
    quantization        Q4_0
```

注意 context length 的差异：
- llama3:8b → 8192 tokens
- qwen2:7b → 32768 tokens（4 倍于 llama3）

## 三、在终端中交互对话

### 3.1 启动 llama3:8b 交互模式

```bash
ollama run llama3:8b
```

进入交互界面后，你会看到：

```text
>>> Send a message (/? for help)
```

试试以下提问：

```text
>>> 用简单的中文解释什么是 Transformer 架构
```

```text
>>> 写一段 Python 代码，实现冒泡排序
```

```text
>>> /bye
```

常用交互命令：

| 命令 | 作用 |
|---|---|
| `/bye` | 退出对话 |
| `/clear` | 清空对话历史 |
| `/show info` | 显示当前模型信息 |
| `/set system <prompt>` | 设置系统提示词 |
| `Ctrl+D` | 退出 |

### 3.2 启动 qwen2:7b 交互模式

```bash
ollama run qwen2:7b
```

用完全相同的问题提问，感受两个模型的回答风格差异。

### 3.3 单次非交互模式（适合脚本调用）

```bash
# 管道输入
echo "用三句话解释神经网络" | ollama run llama3:8b

# 直接传参
ollama run qwen2:7b "请列出 5 个 Python 常用标准库"
```

## 四、模型卡片深度解读

### 4.1 llama3:8b 模型概况

| 属性 | 值 |
|---|---|
| 开发者 | Meta AI |
| 参数量 | 8B（80 亿） |
| 训练数据 | 15T tokens，主要英文 |
| 上下文窗口 | 8192 tokens |
| 优势 | 代码、推理、英文指令跟随 |
| 劣势 | 中文流畅度一般，context 较短 |
| 许可证 | Meta Llama 3 Community License |

llama3:8b 使用了 GQA（Grouped Query Attention）技术，推理速度比 llama2:13b 还快。

### 4.2 qwen2:7b 模型概况

| 属性 | 值 |
|---|---|
| 开发者 | 阿里巴巴 Qwen 团队 |
| 参数量 | 7.6B |
| 训练数据 | 7T tokens，中英双语强化 |
| 上下文窗口 | 32768 tokens（超长） |
| 优势 | 中文表达、长文本处理、指令跟随 |
| 劣势 | 纯英文代码任务略弱于 llama3 |
| 许可证 | Apache 2.0（可商用） |

### 4.3 如何选择模型变体

Ollama 支持拉取同一模型的不同变体：

```bash
# 查看可用 tag
# 访问 https://ollama.com/library/llama3 查看

# 更小量化版本（节省内存）
ollama pull llama3:8b-instruct-q4_K_M

# text completion 模式（非 instruct）
ollama pull llama3:8b-text

# Qwen2 其他尺寸
ollama pull qwen2:1.5b   # 轻量版
ollama pull qwen2:72b    # 旗舰版（需要 48GB+ RAM）
```

## 五、中文任务对比实验

### 5.1 实验设计

对同一组中文任务，分别用 llama3:8b 和 qwen2:7b 回答，主观评分（1-5 分）：

**测试题组**：

```text
1. 解释"道可道，非常道"的含义
2. 写一首关于秋天的七言绝句
3. 用中文总结以下英文段落：[段落省略]
4. 帮我修改这段中文使其更正式：[文本省略]
5. 计算：如果一辆车以 60km/h 行驶 2.5 小时，走了多远？
```

### 5.2 预期观察

| 任务类型 | llama3:8b 表现 | qwen2:7b 表现 |
|---|---|---|
| 中文理解 | 基本能理解，偶有错误 | 理解准确，语感自然 |
| 中文生成 | 有时混入英文词 | 纯中文，文风流畅 |
| 古文解读 | 较弱 | 明显更好 |
| 数学计算 | 一般 | 一般（都是语言模型局限） |
| 代码生成 | 较强 | 够用 |

> **结论**：中文任务优先选 qwen2，英文技术文档或代码任务选 llama3。

### 5.3 快速对比命令

```bash
# 对比中文输出
QUESTION="用中文简单解释量子纠缠"

echo "=== llama3:8b ==="
echo "$QUESTION" | ollama run llama3:8b

echo "=== qwen2:7b ==="
echo "$QUESTION" | ollama run qwen2:7b
```

## 六、模型切换与管理

### 6.1 同时保留多个模型

模型文件存储在 `~/.ollama/models/`，拉取后会一直保留。

```bash
# 查看所有模型
ollama list

# 删除不需要的模型释放空间
ollama rm llama3:8b
```

### 6.2 查看模型层级结构

```bash
# 显示模型的 Modelfile（配置信息）
ollama show --modelfile qwen2:7b
```

```text
# Modelfile generated by "ollama show"
FROM /root/.ollama/models/blobs/sha256-...

TEMPLATE """<|im_start|>system
{{ .System }}<|im_end|>
<|im_start|>user
{{ .Prompt }}<|im_end|>
<|im_start|>assistant
"""

PARAMETER stop "<|im_start|>"
PARAMETER stop "<|im_end|>"
```

这个 Modelfile 揭示了模型的对话模板格式，是后续自定义 system prompt 的基础。

## 小结

1. `llama3:8b` 英文与代码能力强，`qwen2:7b` 中文表达更流畅，context 更长（32K vs 8K）
2. GGUF 是单文件量化格式，Q4_0/Q4_K_M 是内存与质量的最佳平衡点
3. `ollama show` 能快速查看模型参数量、量化方式、context length 等关键指标
4. 同一问题跑两个模型对比，是建立模型直觉最有效的方式
5. 中文任务优先 qwen2，英文技术任务优先 llama3——两者各有所长，无绝对优劣
