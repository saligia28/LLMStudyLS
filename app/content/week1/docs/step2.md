# Step 2: Prompt 工程 - 角色、任务、约束结构深度解析

## 一、Prompt Engineering 核心概念

### 1.1 什么是 Prompt Engineering？
Prompt Engineering（提示词工程）是一门设计和优化与 LLM 交互指令的技术，通过精心构造的提示词，让 AI 产生更准确、更符合预期的输出。

**核心理念：**
- 把 AI 当作一个"新员工"，需要清晰的工作指示
- 越具体的指令 = 越可预测的结果
- 结构化的 Prompt = 更稳定的输出质量

---

## 二、Prompt 的三大核心结构

### 2.1 角色设定（Role）

**作用：** 定义 AI 的身份和专业领域，影响回答的风格和深度。

**语法结构：**
```
你是一个 [专业角色] + [专业特长] + [工作经验/特质]
```

**示例对比：**

❌ **不好的角色设定：**
```
你是一个助手。
```

✅ **好的角色设定：**
```
你是一个拥有 10 年经验的前端开发专家，擅长 React 和性能优化，
善于用简洁的语言解释复杂概念。
```

**常见角色类型：**
- **专业顾问型**：律师、医生、工程师、教师
- **创意工作者**：作家、设计师、营销专家
- **分析专家**：数据分析师、研究员、顾问
- **技能专家**：编程专家、翻译专家、写作专家

**实践技巧：**
1. **具体化**：不要说"程序员"，说"React 前端开发专家"
2. **特质化**：加入性格特点，如"耐心的"、"严谨的"、"富有创造力的"
3. **经验化**：说明经验年限或成就，增强权威性

---

### 2.2 任务描述（Task）

**作用：** 明确告诉 AI 需要做什么，是 Prompt 的核心部分。

**语法结构：**
```
请 [动词] + [对象] + [目标/目的]
```

**关键要素：**

#### 1. 使用明确的动词
- **分析类**：分析、评估、对比、总结
- **创作类**：写作、设计、生成、创建
- **转换类**：翻译、改写、转换、优化
- **解释类**：解释、阐述、说明、教学

#### 2. 提供充足的上下文
```
❌ 简陋的任务：
帮我写代码。

✅ 详细的任务：
为一个电商网站编写用户登录验证功能，需要包括：
- 邮箱和密码验证
- 错误提示
- 密码加密（使用 bcrypt）
- 使用 Express.js 框架
```

#### 3. 明确输入和输出
```
输入：一段包含技术术语的英文文档
任务：翻译成中文
输出：流畅的中文译文，保留专业术语的英文注释
```

---

### 2.3 约束条件（Constraints）

**作用：** 限定输出的格式、长度、风格等，确保结果符合具体需求。

**常见约束类型：**

#### 1. 格式约束
```
- 以 Markdown 表格形式输出
- 使用 JSON 格式返回
- 分点列出，每点不超过 30 字
- 使用代码块包裹代码部分
```

#### 2. 长度约束
```
- 控制在 200 字以内
- 回答分为 3 个段落
- 提供 5 个示例
- 总结不超过 50 词
```

#### 3. 风格约束
```
- 使用简单易懂的语言，避免专业术语
- 采用正式商务风格
- 用幽默轻松的语气
- 面向初学者，需要详细解释
```

#### 4. 内容约束
```
- 不要包含任何代码
- 仅使用 ES6+ 现代 JavaScript 语法
- 避免使用第三方库
- 必须包含具体的数据示例
```

#### 5. 逻辑约束
```
- 先给结论，再给理由
- 按时间顺序排列
- 从简单到复杂逐步说明
- 每个观点必须有数据支撑
```

## 三、完整 Prompt 结构模板

### 3.1 基础模板

```
[角色设定]
你是一个 [具体角色 + 专业特长]。

[任务描述]
请 [动词] [对象]，具体要求：
1. [子任务 1]
2. [子任务 2]
3. [子任务 3]

[约束条件]
- 格式：[输出格式]
- 长度：[字数限制]
- 风格：[语言风格]
- 其他：[特殊要求]

[可选：示例]
例如：[给一个参考示例]
```

### 3.2 高级模板（包含思维链）

```
[角色设定]
你是 [具体角色]。

[任务描述]
请完成以下任务：[具体任务]

[思维过程要求]
请按以下步骤思考：
1. 首先，[分析步骤]
2. 然后，[处理步骤]
3. 最后，[输出步骤]

[约束条件]
- [约束 1]
- [约束 2]

[输出格式]
请按以下格式输出：
```
[格式示例]
```
```

## 四、三种不同风格的 Prompt 实战示例

### 示例 1：专业分析型 - 代码审查助手

**场景：** 需要 AI 审查代码质量并给出改进建议

**完整 Prompt：**

```
【角色设定】
你是一位拥有 15 年经验的高级软件工程师，专注于代码质量和最佳实践。
你曾在 Google 和 Meta 工作，对代码审查有深刻的理解。
你的审查风格：严谨但富有建设性，不仅指出问题，还提供解决方案。

【任务描述】
请审查以下 JavaScript 代码，并提供详细的改进建议。

审查维度：
1. 代码可读性和命名规范
2. 性能优化可能性
3. 潜在的 bug 或边界情况
4. 安全性问题
5. 是否符合现代 ES6+ 最佳实践

【约束条件】
- 按严重程度分类：严重、中等、建议
- 每个问题提供：问题描述 + 原因说明 + 改进代码示例
- 使用 Markdown 格式，代码部分用代码块包裹
- 最后给出改进后的完整代码

【输入代码】
```javascript
function getData(id) {
  var result = null;
  fetch('https://api.example.com/data/' + id)
    .then(res => res.json())
    .then(data => result = data);
  return result;
}
```

【期望输出格式】
## 代码审查报告

### 🔴 严重问题
1. [问题描述]
   - **原因：** ...
   - **改进：**
   ```javascript
   // 改进后的代码
   ```

### 🟡 中等问题
...

### 🟢 建议改进
...

### ✅ 改进后的完整代码
```javascript
// 最终代码
```
```

**为什么这样设计？**
- ✅ 角色具体：明确经验和风格
- ✅ 任务清晰：5 个审查维度
- ✅ 约束详细：分类、格式、结构都有要求
- ✅ 有示例：提供输出格式模板

### 示例 2：创意生成型 - 营销文案创作助手

**场景：** 为产品生成吸引人的营销文案

**完整 Prompt：**

你是一位屡获殊荣的创意文案策划师，曾为苹果、耐克等品牌撰写广告文案。
你的文案特点：简洁有力、富有情感共鸣、善于讲故事。
你深谙消费者心理学，能用最少的文字触动人心。

【任务描述】
为以下产品创作一套完整的营销文案。

产品信息：
- 产品名称：智能水杯 AquaSmart

  * 自动提醒喝水（每小时震动）
  * 水质检测（TDS 传感器）

  * 保温 12 小时

- 价格：299 元
- 品牌调性：科技感 + 关爱健康

【任务要求】
请创作以下内容：
1. 产品 Slogan（一句话，10 字以内）
2. 产品描述（2-3 句话，突出核心价值）
3. 三个核心卖点文案（每个 30 字左右）
4. 一段情感化的场景故事（100 字左右）
5. 行动号召文案（CTA，引导购买）

【约束条件】
- 语言风格：温暖、关怀、不过度营销
- 避免使用："革命性"、"颠覆"等夸张词汇
- 必须包含具体数字或数据
- 每个卖点要对应一个用户痛点
- 故事要有场景感，让人产生代入感

【输出格式】

[一句话]

### 2. 产品描述
[2-3 句话]

...

### 4. 场景故事
[情感化故事]

### 5. 行动号召（CTA）
[引导购买的文案]

---

【创作策略说明】
[简要说明创作思路和用户心理洞察]
```

**为什么这样设计？**
- ✅ 角色专业：文案策划师背景，建立权威感
- ✅ 任务结构化：5 个具体输出物，清晰明确
- ✅ 约束精准：限定风格、禁用词汇、要求数据支撑
- ✅ 输出格式：严格的结构化输出，便于直接使用

### 示例 3：教学辅导型 - 编程概念讲解助手

**场景：** 用通俗易懂的方式讲解编程概念

**完整 Prompt：**

```
【角色设定】
你是一位深受学生喜爱的计算机科学教授，曾获"年度最佳教师"奖。
你的教学特色：
- 善于用生活中的类比解释抽象概念
- 从简单到复杂，循序渐进
- 耐心细致，鼓励学生思考
- 喜欢用图表和代码示例辅助说明

【任务描述】
请向一个刚开始学编程的初学者（零基础）解释以下概念：
**主题：什么是「递归」（Recursion）？**

【教学要求】
请按以下步骤讲解：
1. **生活类比**：用一个生活中的例子引入概念（不要直接讲技术）
2. **概念定义**：用简单的语言给出定义
3. **核心要素**：说明递归的两个关键部分（基准情况 + 递归调用）
4. **代码示例**：提供一个简单的代码示例（JavaScript，带详细注释）
5. **常见错误**：说明初学者容易犯的错误
6. **练习题**：给出一个小练习，帮助理解

【约束条件】
- 语言：中文，避免使用英文专业术语（必要时加中文注释）
- 难度：假设读者完全不懂编程
- 代码：使用 JavaScript，每行代码都要有注释
- 长度：总长度控制在 500 字左右
- 风格：友好、鼓励性，像和朋友聊天

【禁止事项】
- ❌ 不要使用数学公式
- ❌ 不要假设读者了解任何编程概念
- ❌ 不要一次性讲太多，聚焦核心概念
- ❌ 不要使用"显而易见"、"很简单"等表述

【输出格式】
# 轻松理解：递归是什么？

## 🌟 从生活说起
[用生活例子类比，100 字左右]

## 💡 概念解释
[简单定义，50 字以内]

## 🔑 两个关键点
1. **基准情况**：[解释]
2. **递归调用**：[解释]

## 💻 代码示例
```javascript
// 详细注释的代码
```

**代码说明：**
[逐步解释代码做了什么]

## ⚠️ 新手常见误区
- [误区 1]
- [误区 2]

## 🎯 小练习
[一个简单的练习题 + 提示]

---

**🎉 鼓励的话**
[给学习者的鼓励，2-3 句话]
```

**为什么这样设计？**
- ✅ 角色亲和：教师身份，强调教学风格
- ✅ 任务分步：6 步教学流程，符合认知规律
- ✅ 约束细致：明确难度、语言、禁止事项
- ✅ 输出友好：emoji + 结构化，降低学习门槛

## 五、Prompt 优化技巧

### 5.1 从模糊到精确的迭代过程

**版本 1（模糊）：**
```
写一篇关于 AI 的文章。
```

**版本 2（加入角色和任务）：**
```
你是一个科技作家。
写一篇介绍 AI 发展历史的文章。
```

**版本 3（加入约束）：**
```
你是一个科技作家。
写一篇介绍 AI 发展历史的文章。
要求：
- 1000 字左右
- 包含 3 个重要里程碑事件
- 用通俗语言，面向普通读者
```

**版本 4（加入结构和示例）：**
```
你是一个为《科普中国》撰稿的资深科技作家，擅长将复杂技术讲得通俗易懂。

任务：写一篇介绍 AI 发展历史的科普文章。

结构要求：
1. 开头：用一个引人入胜的场景或问题引入
2. 正文：按时间顺序介绍 3 个重要里程碑
   - 1956 年达特茅斯会议
   - 2012 年深度学习突破
   - 2022 年 ChatGPT 发布
3. 结尾：展望未来 3-5 年的发展趋势

约束条件：
- 字数：1000-1200 字
- 语言：通俗易懂，避免专业术语（必要时加解释）
- 风格：客观中立，不过度夸张
- 格式：Markdown，使用合适的标题层级

参考风格：
类似《科学美国人》的文章，既有深度又不失可读性。
```

### 5.2 使用"思维链"提升复杂任务质量

**什么是思维链（Chain of Thought, CoT）？**
让 AI 展示推理过程，而不是直接给答案。

**示例对比：**

❌ **不使用思维链：**
```
计算：如果一个项目团队有 8 人，每人每天工作 6 小时，
项目需要完成 480 小时的工作量，需要多少天？
```

✅ **使用思维链：**
```
计算：如果一个项目团队有 8 人,每人每天工作 6 小时，
项目需要完成 480 小时的工作量，需要多少天？

请按以下步骤思考：
1. 先计算团队每天的总工作时间
2. 再用总工作量除以每天工作时间
3. 给出最终答案

请展示你的计算过程。
```

### 5.3 使用 Few-shot 提供示例

**什么是 Few-shot？**
在 Prompt 中提供几个示例，让 AI 学习你想要的输出格式。

**示例：情感分析**

```
你是一个情感分析专家。请分析以下评论的情感倾向。

【示例】
输入：这个产品真的太好用了，强烈推荐！
输出：正面 | 置信度：95% | 关键词：好用、推荐

输入：质量一般般，价格有点贵。
输出：中性偏负面 | 置信度：70% | 关键词：一般、贵

输入：完全不能用，浪费钱！
输出：负面 | 置信度：98% | 关键词：不能用、浪费钱

【现在请分析】
输入：外观设计不错，但是功能有待改进。
输出：
```

## 八、常见问题 FAQ

### Q1: 如何判断一个 Prompt 是否写得好？

**评判标准：**
1. ✅ **输出稳定性**：多次运行结果一致
2. ✅ **任务完成度**：完全满足需求
3. ✅ **格式符合度**：输出格式符合要求
4. ✅ **质量可预测**：知道大概会得到什么样的回复

**测试方法：**
- 同一个 Prompt 运行 3-5 次
- 检查结果是否一致
- 如果差异很大，说明 Prompt 需要更明确的约束

### Q2: 角色设定真的有用吗？

**答案：非常有用！**

**实验对比：**
```
❌ 不设定角色：
"解释什么是区块链"
→ 输出：技术性强，专业术语多

✅ 设定角色：
"你是一个向小学生讲解科学的老师，请解释什么是区块链"
→ 输出：生动、使用类比、通俗易懂
```

### Q3: Prompt 越长越好吗？

**不一定！** 遵循"必要且充分"原则：

**过短的问题：**
```
写代码
→ AI 不知道写什么代码
```

**过长的问题：**
```
[3000 字的详细需求文档]
→ 信息过载，AI 可能抓不住重点
```

**合适的长度：**
- 简单任务：50-200 字
- 复杂任务：200-500 字
- 超复杂任务：分步提问，而不是一次性给所有信息

### Q4: System 消息和 User 消息有什么区别？

**System 消息（系统提示词）：**
- 定义 AI 的"人设"和基本规则
- 会影响整个对话过程
- 适合放置角色设定和全局约束

**User 消息（用户消息）：**
- 具体的任务和问题
- 每次对话可以不同
- 适合放置具体任务描述

**最佳实践：**
```javascript
const messages = [
  {
    role: 'system',
    content: '你是一个专业的 JavaScript 编程助手，擅长代码优化。'
  },
  {
    role: 'user',
    content: '请优化这段代码：[具体代码]'
  }
];
```

### Q5: 如何让 AI 输出特定格式（如 JSON）？

**方法 1：明确要求格式**
```
请以 JSON 格式输出，包含以下字段：
- name: 姓名
- age: 年龄
- skills: 技能列表（数组）

示例：
{
  "name": "张三",
  "age": 25,
  "skills": ["Python", "JavaScript"]
}
```

**方法 2：使用 Few-shot**
```
请将以下信息转换为 JSON 格式。

示例 1：
输入：李明，28 岁，会 Java 和 Go
输出：{"name": "李明", "age": 28, "skills": ["Java", "Go"]}

示例 2：
输入：王芳，32 岁，会 Python
输出：{"name": "王芳", "age": 32, "skills": ["Python"]}

现在请转换：
输入：陈伟，26 岁，会 C++ 和 Rust
输出：
```

**方法 3：使用 response_format（OpenAI 新特性）**
```javascript
const response = await client.chat.completions.create({
  model: 'gpt-4-turbo-preview',
  messages: [...],
  response_format: { type: 'json_object' }
});
```

### Q6: 如何处理 AI 输出不符合预期的情况？

**迭代优化流程：**

1. **分析问题**
   - 是格式不对？→ 加强格式约束
   - 是内容不准确？→ 提供更多上下文
   - 是风格不符？→ 调整角色设定
   - 是理解错误？→ 任务描述更明确

2. **加入示例**
   - 提供正面示例（期望的输出）
   - 提供反面示例（不期望的输出）

3. **分步提问**
   - 复杂任务拆分成多个简单步骤
   - 使用思维链引导推理过程

4. **调整参数**
   - 降低 `temperature` 提高稳定性
   - 增加 `max_tokens` 避免输出被截断

## 九、进阶学习资源

### 9.1 Prompt Engineering 技术

**1. Chain of Thought（思维链）**
- 让 AI 展示推理过程
- 适合复杂问题和数学题
- 关键词："让我们一步步思考"、"请展示推理过程"

**2. Few-shot Learning（少样本学习）**
- 在 Prompt 中提供 3-5 个示例
- AI 会学习示例的模式
- 适合格式转换、分类任务

**3. Zero-shot Learning（零样本学习）**
- 不提供示例，直接下任务
- 适合通用任务
- 依赖清晰的指令

**4. ReAct（推理+行动）**
- 结合推理和工具调用
- 适合需要外部信息的任务
- 进阶话题，建议后续学习
